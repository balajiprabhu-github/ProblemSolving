Two Pointers — Cheat Sheet

Patterns · Triggers · Invariants · Traps · Interview Tips

Core Idea
What is it?
Two Pointers
Use two indices to scan an array (or two arrays) simultaneously — eliminating the need for a nested loop by moving pointers deliberately based on a condition.
Brute force checks all pairs: O(n²)
Two pointers collapses that to: O(n)
When to reach for it
Universal Triggers
  • Sorted array + pair/triplet sum
  • "Find two elements that…"
  • "In-place" modification of array
  • Compare two strings/sequences
  • "Unique" combinations / no duplicates
  • Shrink a window from both ends
Golden Rule
Every pointer move must be justified
Never move a pointer arbitrarily. Ask: "Does moving this pointer keep me on track toward the answer?" If you can't answer — rethink.
Pattern 1 — Opposite Ends (Converging)
Setup
left = 0
right = n−1
Both pointers start at opposite ends and move toward each other. Loop runs while left < right.
Trigger
When to use
  • Sorted array + pair that meets a condition
  • Palindrome check
  • Max area / water trapped problems
  • "Find two numbers that sum to X"
Mnemonic: Two people walking toward each other on a number line.
Invariant
Why it works
Array is sorted → moving left++ increases sum, moving right-- decreases sum. Every move discards an entire row/column of the brute-force matrix.
Breaks if: array is not sorted — movement has no guaranteed direction.
Trap
Moving the wrong pointer
In Container With Most Water — moving the taller wall never helps. Width shrinks AND height is still capped by the shorter side.
Always move the limiting factor.
Pattern 2 — Fast / Slow (Same Direction)
Setup
slow = 0
fast = 0 or 1
Both start near the beginning and move in the same direction. fast scouts ahead; slow marks the "write position".
Trigger
When to use
  • Remove/partition elements in-place
  • Detect cycle in a linked list / array
  • "Move all X to the end"
  • "Remove duplicates in-place"
  • Find duplicate without extra space
Invariant
Slow = last valid position
Everything at 0..slow is already "processed and correct". fast scans the rest looking for the next valid element. When found → slow++ then place it there.
Floyd's Cycle: Phase 2 — one pointer from index 0, one from meeting point. They meet at the cycle entrance = duplicate.
Trap
Floyd's Phase 2 start
Phase 2: second pointer starts at index 0 — NOT at the meeting point. This is the #1 mistake on LC 287.
Pattern 3 — Two Sequences
Setup
i → string/array 1
j → string/array 2
One pointer per sequence. They move independently based on match or mismatch.
Trigger
When to use
  • Compare two strings/arrays
  • "Is X a subsequence of Y?"
  • Merge two sorted arrays in-place
  • Process two strings with special characters
Key Insight
Forward vs Reverse
Forward (Is Subsequence): i advances only on match — j always advances.

Reverse (Backspace Compare, Merge Sorted Array): process from the back when a character/element affects what came before it (backspace, overwrite risk).
Rule: if future characters invalidate past ones → go backward.
Trap — LC 88
Fill from front = overwrite bug
Merging into nums1 from the front overwrites unread values. Always fill from the back into the empty slots.
Pattern 4 — kSum / Partition
kSum Structure
k−2 loops + Two Pointers
3Sum: 1 loop + 2 pointers → O(n²)
4Sum: 2 loops + 2 pointers → O(n³)
kSum: k−2 loops + 2 pointers → O(n^(k−1))
Always sort first. Reduce k down to 2Sum (two pointers).
Duplicate Skipping — Know All 4 Levels
Skip guards per pointer
i > 0 && nums[i] == nums[i−1] → skip anchor i
j > i+1 && nums[j] == nums[j−1] → skip anchor j
left < right && nums[left] == nums[left−1] → skip after match
left < right && nums[right] == nums[right+1] → skip after match
Each anchor's guard: "have I moved past my starting position?"
i starts at 0 → guard is i > 0
j starts at i+1 → guard is j > i+1
After a match
Always move BOTH pointers inward
left++ and right--
Standing still after a match = infinite loop. This is the #1 bug in 3Sum/4Sum.
Dutch National Flag
3-pointer partition
0 → swap(low, mid), low++, mid++
1 → mid++
2 → swap(high, mid), high-- (mid stays!)
mid stays on 2-swap because swapped value is unknown.
Interview Reminders
Complexity
Always state these
  • Sort first? → O(n log n)
  • Single scan → O(n)
  • 1 outer loop + 2 pointers → O(n²)
  • 2 outer loops + 2 pointers → O(n³)
  • Two pointers → O(1) space (vs hash map O(n))
Overflow Trap
Cast to Long for kSum
Int.MAX_VALUE ≈ 2.1 × 10⁹
4 values of 10⁹ = 4 × 10⁹ → overflow!
Fix: nums[i].toLong() + nums[j] + ...
One toLong() propagates the whole expression.
Gut Check
3 Questions Before You Code
  • Is the array sorted? (or should I sort?)
  • Am I moving the right pointer?
  • Did I handle duplicates at all levels?
